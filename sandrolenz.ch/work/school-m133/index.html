Webshop

A school project for M133, where the goal was to build a working webshop with the help of PHP Sessions.

M133

Web-Applikation mit Session-Handling realisieren

Our task in this module was to create a fictional webshop based on a template. The shop should include a product page that is dynamically loaded from a database, a detail page for each product as well as a shopping cart and the ability to place an order which is then stored in the database. Adding to that, we also had to implement a login system and an additional page to manage and create new products.

The interface was built using HTML and CSS. Storing the shopping cart and login status was made possible with PHP sessions. I also used JavaScript (jQuery) for some parts of the page. The database was written in MySQL.

You can view more info about the module on the "Modulbaukasten" page. (in German, French or Italian)

The shop

The reason I'm selling muffins is because they are a little insider with some of my friends. So as soon as I heard that we were going to build a shop, it was very clear that it would have to sell muffins.

We started with some smaller tasks before we began work on the actual shop to get into the topic. Thankfully, I didnt have a lot of problems with these tasks since I was already familiar with HTML, CSS & PHP.

After that we had to create the database scheme, a sitemap, wireframes and mockups for our shop.

Login system

To login, one just needs to click the profile icon in the header and enter their login data. Said data is then checked against the hashed password stored in the database. If it is correct, the user will be redirected to the product management page and the login state is saved using the PHP-Session.

If the user is not logged in but tries to access the management page, a simple piece of code will redirect them to the login page.
